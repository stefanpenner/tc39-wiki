Display Name

This is a rough proposal to add an API to obtain display names for locale and its components such as languages, regions and scripts.

Use cases

  • UI elements (menu, status) on web-based OS
  • Language / region / script pickers in web pages, web apps, browser extensions ; wikipedia language list, Chrome extension for per-script font selection

Proposed API

 
 
 
/**
 * Create a DisplayName object for |displayLocale|
 * displayLocale {LocaleList|String} Locale in which names are displayed.
 * isDialect : determines which form to use, standard ('English (United Kingdom)') or dialect ('British English'). The default : false 
 * @constructor
 */
 
Intl.DisplayName(displayLocale, isDialect);
 
 
/**
 * Create a DisplayName object for the default locale. 
 * isDialect : determines which form to use, standard ('English (United Kingdom)') or dialect ('British English'). The default : false 
 * @constructor
 */
Intl.DisplayName(isDialect);
 
Intl.DisplayName.prototype.localeName(locale);
 
Intl.DisplayName.prototype.languageName(lang);
 
Intl.DisplayName.prototype.regionName(region);
 
Intl.DisplayName.prototype.scriptName(script);
 

Discussion

  • How about variant names such as German orthography?
  • How about key-value pairs in locale names?
  • Are there other options than ‘isDialect’?
  • Do we want to support different formatting types; ‘long/official’ name, ‘short/colloquial name’, ‘abbreviation’ (e.g. Republic of Korea, South Korea, ROK)
  • Is |isDialect| relevant to all methods? Yes, it is unless we add ‘variant’ or ‘key-value’.
  • Is |isDialect| supposed to distinguish between ‘South Korea’ and ‘Korea, South’ (or ‘Korea (South)’) ?
