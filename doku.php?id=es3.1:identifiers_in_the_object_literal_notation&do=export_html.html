Description: The ES3 Specification does not allow the use of reserved words as identifiers in the key/value pairs in the literal object notation. There is no reason to have this restriction.

The following statement is currently illegal, producing syntax errors. It should be legal.

var object = {function: false, goto: “home”, var: 27};

Relevant Goal: 1
Relevant Design Principle: 2a, 2b
Relevant ES4 proposal: reserved words
